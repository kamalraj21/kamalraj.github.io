Installing iManager

NetIQ iManager provides a global view of your network from a single browser-based tool, allowing you to proactively assess and respond to changing network demands.

Using iManager, you can administer NetIQ eDirectory and other NetIQ and third-party products, including Novell Open Enterprise Server, NetIQ Identity Manager, Novell Audit, and BorderManager. This centralized management system minimizes administrative redundancy and overhead, reducing time and costs.

This section provides information about:

  • Server-based and client-based versions of iManager
  • Self-signed certificates

Server-Based and Client-Based Versions of iManager

The traditional server-based version of iManager 2.7.7, commonly called iManager, installs on a server to access an eDirectory tree. The client-based version, called iManager Workstation, installs on a client workstation.

Use the following guidelines to determine which version best fits your environment.

iManager Workstation

Use iManager Workstation if you have a single administrator who manages eDirectory from a dedicated client workstation, or if you have multiple administrators who require customized plug-ins.

You must install plug-ins on each administrator's client workstation, as iManager plug-ins do not automatically synchronize between iManager instances. Key features of iManager Workstation include:

  • Self-contained deployment: Setup is fast and simple.
  • Automatic resource management: It automatically starts and stops required resources during loading and unloading.
  • Cross-platform support: It installs and runs on various Linux and Windows client workstations.
  • Independent operation: It has no dependencies on server-based iManager and coexists with other iManager versions on your network.

iManager Server

If you manage eDirectory from multiple client workstations or have multiple administrators, install iManager Server to ensure availability from any connected workstation. Additionally, you only need to install customized plug-ins once per iManager Server.

Note

iManager 2.7.7 Patch 11 installs the iManager 2.7.7 base along with the patch.

Application Servers

iManager Server supports Tomcat 7.0.81.

Caution

Do not install the standalone iManager package on a Tomcat server that runs other applications.

Browsers

iManager Server supports the following browsers:

  • Apple Safari 8.0 and 7.0.6
  • Google Chrome 38 through 22
  • Internet Explorer 11, 10 (Normal and Compatibility modes), 9, and 8
  • Mozilla Firefox 33 through 9.0.1, and 4.0.1

Directory Services

iManager supports eDirectory 8.8 and later Support Packs.

Configuring Certificates on Linux

Standalone iManager installations include a temporary, self-signed certificate for Tomcat. It expires in one year. Because OpenSSL does not recommend self-signed certificates for production, use this certificate only to get your system up and running securely immediately after installation.

Tomcat uses the Java KeyStore (JKS) format by default. The keytool utility cannot import a private key; it only uses self-generated keys. If you use eDirectory, use Novell Certificate Server to generate and manage certificates securely.

Note

These instructions do not apply to OES Linux, which installs both Tomcat and Apache. To replace certificates on OES Linux, see the OES Implementation Guide.

Follow these steps to create a key pair in eDirectory, export the keys via a PKCS#12 file on Linux, and modify the Tomcat configuration to use the new file.

Prerequisites

Ensure you have access to the following paths:

  • Temporary key pair: /var/opt/novell/novlwww/.keystore
  • Trusted roots: /opt/novell/jdk1.7.0_25/jre/lib/security/cacerts
  • Tomcat configuration: /etc/opt/novell/tomcat7/server.xml

Procedure

  1. In iManager, create a new server certificate:

    Go to Novell Certificate Server > Create Server Certificate. Select the appropriate server, specify a nickname, and accept the default settings.

  2. Export the server certificate to the Tomcat home directory (/var/opt/novell/novlwww).

  3. Convert the .pfx file to a .pem file:

    openssl pkcs12 -in newtomcert.pfx -out newtomcert.pem

    Enter the certificate password from the previous step, and then define a password for the new .pem file.

  4. Convert the .pem file to a .p12 file:

    openssl pkcs12 -export -in newtomcert.pem -out newtomcert.p12 -name "New Tomcat"

    Enter the password specified in Step 3, and define a password for the new .p12 file.

  5. Stop the Tomcat service:

    /etc/init.d/novell-tomcat7 stop

  6. Open /etc/opt/novell/tomcat7.0.42/server.xml in a text editor and add the keystoreType, keystoreFile, and keystorePass attributes to the Connector configuration. For example:

    <Connector
  className="org.apache.coyote.tomcat7.CoyoteConnector"
  port="8443" minProcessors="5" maxProcessors="75"
  enableLookups="true"
  acceptCount="100" debug="0" scheme="https" secure="true"
  useURIValidationHack="false" disableUploadTimeout="true">
  <Factory
    className="org.apache.coyote.tomcat7.CoyoteServerSocketFactory"
    clientAuth="false" protocol="TLS" keystoreType="PKCS12"
    keystoreFile="/var/opt/novell/novlwww/newtomcert.p12"
    keystorePass="password" />
</Connector>

  7. Set ownership and permissions for the .p12 file:

    chown novlwww:novlwww newtomcert.p12
chmod 654 newtomcert.p12

  8. Start the Tomcat service:

    /etc/init.d/novell-tomcat7 start
