Configuring an AI Agent for Citizen Services using Salesforce Agentforce

Product: Salesforce Public Sector Solutions
Feature: Agentforce
Version: Spring '24 Release

Overview

Salesforce Agentforce enables government agencies and public sector organizations to build autonomous AI agents that handle routine citizen inquiries, automate service requests, and triage complex issues. By configuring an AI Agent, you can provide 24/7 constituent support while reducing case volume for human representatives.

This guide explains how to configure a new AI Agent to assist citizens with municipal service requests (such as reporting a pothole or requesting a permit) using the Agent Builder.

Prerequisites

Before you begin, ensure you have the following:

  • System Administrator permissions in your Salesforce Public Sector org.
  • The Agentforce Service feature license enabled.
  • Preconfigured Data Cloud ingestion streams mapped to your public service knowledge base.

Creating the Citizen Services AI Agent

Use the Agent Builder to define the AI Agent's role, connect it to your knowledge base, and specify its conversation tones and boundaries.

To create a new AI Agent:

  1. From Setup, in the Quick Find box, enter Agentforce, and then select Agent Builder.
  2. Click New Agent.
  3. In the Provide a Name field, enter Citizen Services Assistant.
  4. In the Description field, enter An AI Agent designed to help constituents navigate public service requests and knowledge articles.
  5. Click Next.

Configuring the Agent Profile and Tone

The agent profile dictates how the AI formulates responses. Because this agent represents a government entity, its tone must be professional, helpful, and objective.

To configure the agent's tone and instructions:

  1. In the Agent Builder workspace, click the Profile tab.
  2. Under System Prompt, provide explicit instructions for the language model. For example, enter:
    "You are a helpful, professional representative of the municipal government. Answer constituent questions using only the provided knowledge base. Do not speculate or make promises on behalf of the agency. If you cannot find the answer, politely offer to escalate the inquiry to a human agent."
  3. In the Tone dropdown list, select Professional and Empathetic.
  4. Click Save.

Assigning Knowledge Actions

To ensure the AI Agent provides accurate answers, you must bind it to your Data Cloud knowledge repository.

Important

Knowledge Actions override the underlying large language model's (LLM) training data. The agent will only draw conclusions from the Data Cloud objects you specify here.

To assign knowledge actions to the agent:

  1. Click the Actions tab.
  2. Click Add Action, and then select Knowledge Retrieval.
  3. In the Select Data Space field, choose Public_Sector_Knowledge.
  4. In the Data Model Objects (DMOs) list, select Public Service Articles and Permit Guidelines.
  5. Click Done.

Defining Escalation Rules

When the AI Agent encounters a scenario it cannot resolve, it must smoothly transition the conversation to a human constituent service representative.

To configure the escalation protocol:

  1. Click the Settings tab.
  2. Under the Handoff section, toggle Enable Human Escalation to the active position.
  3. In the Omni-Channel Queue dropdown list, select Tier 1 Support (Public Services).
  4. In the Escalation Message field, enter Transferring you to a representative who can assist with this specific request. Please hold.
  5. Click Save & Publish.

Verifying the Configuration

Before deploying the agent to a public-facing Experience Cloud site, test its responses internally using the Agentforce Sandbox.

To test the AI Agent:

  1. From the top-right corner of the Agent Builder, click Preview.
  2. In the chat window, type: How do I report a pothole on Main Street?
  3. Verify that the AI returns the correct procedure mapped from the Public Service Articles DMO.
  4. Type an off-topic query (e.g., Can you calculate my income tax?).
  5. Verify that the AI apologizes and triggers the escalation message to the Omni-Channel queue.

Once verification is successful, your AI Agent is ready for deployment in production.
