Value Proposition

Embracing the Docs-as-Code approach, I integrate documentation into the development workflow using version control, automation, and modern tooling. This ensures consistency, accuracy, and seamless collaboration with development teams.

  • Version control: Utilize Git to manage documentation, enabling collaborative editing, versioning, and review processes.
  • Automated builds: Integrate documentation builds into CI/CD pipelines for automated testing and publication.
  • Markdown and Sphinx: Create and maintain documentation in Markdown and Sphinx, supporting modular and easily accessible content.

Tools & Technologies

Core tools used in my Docs-as-Code practices include:

  • Version Control: Git, GitHub, GitLab
  • Documentation Frameworks: Sphinx, MkDocs
  • CI/CD Integration: Jenkins, GitHub Actions
  • Markdown Editors: Visual Studio Code, Typora
